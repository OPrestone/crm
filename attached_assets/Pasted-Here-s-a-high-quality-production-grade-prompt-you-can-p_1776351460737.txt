## 🔧 REPLIT AI PROMPT: Build a Multitenant Laravel CRM Platform

**Prompt:**

Build a **world-class, enterprise-grade, multitenant CRM platform** using **PHP (Laravel latest stable version)** with a modular, scalable architecture.

---

### 🧱 CORE ARCHITECTURE REQUIREMENTS

* Framework: **Laravel (latest version)**
* Architecture: **MVC strictly enforced**

  * All database queries MUST be inside **Controllers or Service Classes (no queries in Blade views)**
* Multitenancy:

  * Support **database-per-tenant OR schema-based tenancy**
  * Tenant isolation and secure data separation
  * Tenant onboarding (signup → auto provisioning)
* Authentication:

  * Laravel Breeze or Jetstream (customized)
  * Role-Based Access Control (RBAC)
  * Permissions system (fine-grained access levels)
* API:

  * RESTful API support (for mobile/external integrations)
* Frontend:

  * Blade + Bootstrap (or Tailwind if needed)
  * All **CSS and JS must be stored in `/public/assets`**
  * No inline CSS/JS except minimal dynamic cases

---

### ⚡ PERFORMANCE & OPTIMIZATION

* Implement:

  * Query optimization (Eloquent relationships, eager loading)
  * **Lazy loading** for heavy datasets (pagination, infinite scroll)
  * **Caching system** (Redis or file cache fallback)

    * Cache frequently accessed data (users, dashboards, reports)
  * Queue system for background jobs (emails, notifications)
* Use:

  * Laravel Jobs, Events, Listeners
  * Rate limiting for APIs

---
